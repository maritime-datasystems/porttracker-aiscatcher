Based on my exploration of the full AIS-catcher repository, here's the answer:

Server-Side Code in the AIS-catcher Repository
No, the server/receiver code for the aiscatcher.org community hub is NOT in this repository.

What's in the AIS-catcher Repository (Client-Side Only)
The repository contains client-side components only:

Component	Description
TCPClientStreamer
Sends data TO aiscatcher.org:4242
TCPlistenerStreamer
Local TCP server to serve NMEA to local clients
UDPStreamer
Send NMEA/JSON via UDP
HTTPStreamer
POST data to HTTP endpoints
PostgreSQL.cpp
Write to YOUR local database
WebViewer
Local web UI for YOUR station

If You Want to Build Your Own Receiver/Aggregator
You can use the existing components to build a private aggregator:

bash
# Station 1 sends to your server
AIS-catcher -P your-server-ip 4242 JSON on
# Your server receives and stores
AIS-catcher -x 0.0.0.0 4242 -D "postgresql://user:pass@localhost/ais"
The pieces exist for you to build your own server-side, but the actual aiscatcher.org infrastructure is maintained privately by jvde-github.


4. What Gets Shared
The data is sent as JSON-wrapped NMEA messages (COMMUNITY_HUB format) with:

Raw NMEA sentences
Metadata (signal strength, timestamps, receiver info)
Your station's UUID for identification
5. Key Points
Aspect	Details
Connection direction	Outbound from your device (no port forwarding needed)
Fixed IP required?	No - your device connects to the server's fixed IP
Protocol	TCP with keep-alive
Data format	JSON with embedded NMEA
Authentication	UUID/Sharing Key embedded in each message
Firewall	Only needs outbound access to port 4242
Summary
The sharing key acts as your station's authentication token - it's embedded in every message you send to the server via TCP. The server at 185.77.96.227:4242 receives these messages and associates them with your registered station. Since it's an outbound connection, you don't need a fixed IP or special network configuration - you just need to be able to make outbound TCP connections.
